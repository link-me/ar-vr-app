# ROADMAP (RU)

Цели и план развития проекта AR-VR App — Snake3D (Godot).

## Сделано

— Базовое движение змейки с постоянным вперёд и плавным поворотом.
— Буфер пути головы, равномерный выбор точек для хвоста.
— Сглаживание положения/поворота сегментов (follow/look rates).
— Еда (Food) увеличивает хвост, шипы (Spikes) вызывают Game Over.
— Экран Game Over с кнопкой Restart и клавишей `E`.
— Камера третьего лица: дистанция/высота/сглаживание.
— Встроенная консоль: `restart`, `speed`, `turn`, `auto_forward`, камера‑параметры, `segments_*`, `cubes`, `console`, `console_height`.
— Веб‑экспорт (WASM) и простой локальный сервер.

## Ближайшие задачи

— Счёт/очки и базовый HUD (UI улучшения).
— Мобильные/тач‑управления (экранные кнопки, жесты).
— Звуки и базовые VFX (съедание, столкновение, game over).
— Столкновения с границами поля (варианты: отскок/поражение).
— Опциональное клеточное движение (grid mode).
— Консоль: история команд, автодополнение, помощь по синтаксису.
— Сохранение настроек (скорость, камера, авто‑вперёд) в файл.
— Профили производительности, оптимизация под слабые устройства.
— Документация по экспорту/CI для автоматической сборки Web билда.

## Среднесрочные задачи

— Генерация уровней/препятствий, прогрессия сложности.
— Многопользовательские режимы (локально/онлайн, исследование вариантов).
— Таблица рекордов (локально/в сети).
— Расширенные эффекты: постпроцесс, частицы с оптимизацией под Web.

## Эксперименты AR/VR

— Изучение Godot XR (OpenXR) для возможного VR режима.
— UI/UX адаптация под HMD: камера, управление, комфорт.
— Производительность и качество рендера для XR устройств.

## Риски и допущения

— WebAssembly: ограничения по памяти/производительности в браузерах.
— Мобильные браузеры: сборка/управление/производительность могут отличаться.
— XR: поддержка зависит от устройств и качества драйверов.

## Подход к качеству

— Авто‑тестирование сборки (минимум smoke‑тест через статический сервер).
— Ручные сценарии: управление, еда/шипы, рестарт, консольные команды.
— Профилирование в браузере (FPS, GC, сеть) для тонкой настройки.
